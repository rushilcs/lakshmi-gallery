# Lakshmi Gallery

Photo gallery app for events: **admin** uploads and curates galleries; **guests** view and interact via a share link. Stack: **React + Vite** (`client/`), **Express + Drizzle + Postgres** (`server/`), **S3** for media, optional **SQS** for async jobs, **Sharp** for thumbnails.

## Repository layout

| Path        | Role |
|------------|------|
| `client/`  | SPA: public gallery (`/g/:share_token`), admin UI (`/admin`, …) |
| `server/`  | API, auth, presigned uploads, image pipeline, DB |

## Requirements

- **Node.js** (Vite 7 suggests 20.19+ or 22.12+)
- **PostgreSQL** and a `DATABASE_URL`
- **AWS** account with S3 (and optional CloudFront, Rekognition, SQS) for full production behavior

## Environment (server)

Copy or export variables as needed; the server validates config at startup (`server/src/config.ts`).

| Variable | Notes |
|----------|--------|
| `DATABASE_URL` | **Required** — Postgres connection string |
| `ADMIN_PASSWORD` | Admin login (omit only for local/dev with documented default) |
| `SESSION_SECRET` | Cookie signing |
| `AWS_REGION`, `S3_BUCKET` | Object storage |
| `CLOUDFRONT_*` | Optional signed CDN URLs |
| `SQS_QUEUE_URL` | Optional; async worker queue in production |
| `ALLOWED_ORIGINS` | Comma-separated CORS origins (default includes `http://localhost:5173`) |
| `PUBLIC_APP_URL` | Optional absolute app URL |

## Quick start (local)

1. **Database** — from `server/`:

   ```bash
   npm install
   npm run db:migrate
   ```

2. **API** — from `server/`:

   ```bash
   npm run dev
   ```

   Default port **4000** (see `PORT`).

3. **Worker** (thumbnails / processing) — separate terminal, `server/`:

   ```bash
   npm run worker
   ```

4. **Client** — from `client/`:

   ```bash
   npm install
   npm run dev
   ```

   Vite proxies `/api` → the API server on port 4000. The client always uses same-origin **`/api`** (no hardcoded `http://` URLs) so admin login works over HTTPS in production; CloudFront forwards `/api/*` to the ALB (see `infra/lib/stack.ts`).

## Routes (client)

| Path | Purpose |
|------|---------|
| `/admin` | Admin login, gallery list, **New gallery** (creates and opens the new gallery) |
| `/admin/gallery/:id` | Manage gallery: Content, **Folders**, Settings, Watermark, Publish |
| `/admin/gallery/:id/upload` | Upload photos (files / folder tree) |
| `/admin/gallery/:id/preview` | Logged-in preview of the guest experience + **album order** (save updates sidebar order) |
| `/g/:share_token` | Public gallery for guests |

## Terminology

- **Admin — “Folders”** — Curated groups you create under **Manage → Folders**. You can rename (✎) and assign photos; order is controlled from **Preview** (“Save album order”).
- **Guest UI — “Albums”** — The same groups appear in the sidebar as **Albums** (no per-folder photo counts in the nav).

Upload ZIP/folder structure still surfaces as path names in the sidebar alongside curator albums.

## Notable API (admin, cookie session)

- `POST /api/admin/login` / `logout` / `GET session`
- `POST /api/admin/galleries` — create (returns `{ gallery }`)
- `PUT /api/admin/galleries/:galleryId/folders/order` — body `{ folder_ids: string[] }` (every folder id exactly once)
- `POST /api/admin/galleries/:galleryId/rebuild-thumbnails` — enqueue thumb regeneration (see below)

## Static assets (client)

- Tab icon: `client/public/favicon.svg` (replace with your official black logo if you have a vector).
- In-app sidebar logo: `GalleryViewer` expects **`/logo.png`** — add `client/public/logo.png` if you want it to load.

## Thumbnail quality (1200px long edge)

Thumbnails are generated by the worker with:

- `rotate()` (EXIF-aware)
- `resize({ width: 1200, height: 1200, fit: "inside", withoutEnlargement: true, kernel: sharp.kernel.lanczos3 })`
- light `sharpen(0.5)`
- `jpeg({ quality: 85, chromaSubsampling: "4:4:4", mozjpeg: true })`

Original uploads are stored as-is. Preview behavior is unchanged.

### Rebuild thumbnails for an existing gallery

Admin-only endpoint:

- `POST /api/admin/galleries/:galleryId/rebuild-thumbnails`

Example:

```bash
curl -X POST \
  -b "lakshmi_admin_session=<YOUR_ADMIN_COOKIE>" \
  "https://<your-domain>/api/admin/galleries/<galleryId>/rebuild-thumbnails"
```

The endpoint is idempotent and safe to run multiple times. It enqueues regenerate jobs per image and logs start/progress/summary.

### Verify thumbnail usage

1. Upload a new image.
2. Confirm thumb object exists in S3 and inspect metadata/size:

   ```bash
   aws s3 ls "s3://$S3_BUCKET/galleries/<galleryId>/thumb/" --recursive
   aws s3api head-object --bucket "$S3_BUCKET" --key "galleries/<galleryId>/thumb/<photoId>.jpg"
   ```

3. In browser DevTools (Network), confirm gallery grid requests are thumb URLs (path contains `/thumb/` and `.jpg`).
4. Confirm visual quality in the grid at current tile sizes.
